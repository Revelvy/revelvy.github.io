Spring Bean 的生命周期

流程

Spring会扫描指定包下面的普通 Java 类，将其变成 beanDefinition 对象，然后创建 Spring Bean 对象。

为什么不直接使用对象的class对象来创建bean呢？

因为在class对象仅仅能描述一个对象的创建，它不足以用来描述一个Spring bean，而对于是否为懒加载、是否是首要的、初始化方法是哪个、销毁方法是哪个，这个Spring中特有的属性在class对象中并没有。

普通 Java 类 -> Bean 定义对象

抽象应用上下文中的 refresh( ) 方法

AbstractApplicationContext.java

@Override
public void refresh() throws BeansException, IllegalStateException {
    synchronized (this.startupShutdownMonitor) {
        // 准备上下文
        prepareRefresh();
        // 告诉子类刷新内部 Bean 工厂
        ConfigurableListableBeanFactory beanFactory = obtainFreshBeanFactory();
        // 准备 Bean 工厂
        prepareBeanFactory(beanFactory);
        try {
            // 允许在上下文子类中对 Bean 工厂进行后置处理
            postProcessBeanFactory(beanFactory);
            // 调用在上下文中已被注册为 Bean 的工厂处理器 
            invokeBeanFactoryPostProcessors(beanFactory);
            // 注册 Bean 后置处理器，用来拦截 Bean 创建
            registerBeanPostProcessors(beanFactory);
            // 初始化消息源
            initMessageSource();
            // 初始化事件多播器
            initApplicationEventMulticaster();
            // 初始化特定上下文子类中的其他特殊 bean。
            onRefresh();
            // 检查并注册侦听器 bean 
            registerListeners();
            // 实例化所有剩余的（非延迟初始化）单例
            finishBeanFactoryInitialization(beanFactory);
            // 最后一步：发布相应的事件
            finishRefresh();
        }
        catch (BeansException ex) {
            if (logger.isWarnEnabled()) {
                logger.warn("Exception encountered during context initialization - " +
                            "cancelling refresh attempt: " + ex);
            }
            // 销毁已经创建的单例以避免悬空资源
            destroyBeans();
            // Reset 'active' flag.
            cancelRefresh(ex);
            // 将异常传播给调用者
            throw ex;
        }
        finally {
            // 重置 Spring 核心中的常见内省缓存，因为可能不再需要单例 bean 的元数据
            resetCommonCaches();
        }
    }
}
调用 Bean 工厂的后置处理器 invokeBeanFactoryPostProcessors( )

1⃣ 先调用 Bean 定义对象注册的后置处理器（子接口）的 Bean 定义对象的注册方法

BeanDefinitionRegistryPostProcessor.java

public interface BeanDefinitionRegistryPostProcessor extends BeanFactoryPostProcessor {
    // 扫描指定包下的带有组件注解的普通 Java 类，将其变成 Bean 定义对象，放到一个 Spring 的 Map 中
	void postProcessBeanDefinitionRegistry(BeanDefinitionRegistry registry) throws BeansException;
}

2⃣ 再调用 Bean 工厂的后置处理器（父接口）的 Bean 工厂后置处理器方法

BeanFactoryPostProcessor.java

@FunctionalInterface
public interface BeanFactoryPostProcessor {
	void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory) throws BeansException;
}
注册 Bean 工厂的后置处理器 registerBeanPostProcessors( )

根据实现了 PriorityOrdered、Ordered 接口，排序后注册所有的 Bean 后置处理器，主要用于 Spring Bean 创建时，执行这些后置处理器的方法，这也是 Spring 中提供的扩展点，在此处能插手 Spring bean 的创建过程。

结束 Bean 工厂的后置处理器 finishBeanFactoryInitialization( )

结束 Bean 工厂的初始化，完成非懒加载的Spring bean的创建工作。

Bean 定义对象 -> Spring Bean 对象

流程

4个后置处理器接口 + 8个后置处理器的方法 贯穿在 Bean 对象的实例化、属性赋值、初始化、销毁的过程中。

4个后置处理器接口

BeanPostProcessor
public interface BeanPostProcessor {
	@Nullable
	default Object postProcessBeforeInitialization(Object bean, String beanName) throws BeansException {
		return bean;
	}
	@Nullable
	default Object postProcessAfterInitialization(Object bean, String beanName) throws BeansException {
		return bean;
	}
InstantiationAwareBeanPostProcessor

实例化已知的 Bean 后置处理器。

内部提供了3个方法，再加上BeanPostProcessor接口内部的2个方法，所以实现这个接口需要实现5个方法。

作用：目标对象的实例化过程中需要处理的事情，包括实例化对象的前后过程以及实例的属性设置

public interface InstantiationAwareBeanPostProcessor extends BeanPostProcessor {
	@Nullable
	default Object postProcessBeforeInstantiation(Class<?> beanClass, String beanName) throws BeansException {
		return null;
	}
	default boolean postProcessAfterInstantiation(Object bean, String beanName) throws BeansException {
		return true;
	}
	@Nullable
	default PropertyValues postProcessProperties(PropertyValues pvs, Object bean, String beanName)
			throws BeansException {
		return null;
	}

在org.springframework.beans.factory.support.AbstractAutowireCapableBeanFactory#createBean()方法的Object bean = resolveBeforeInstantiation(beanName, mbdToUse);方法里面执行了这个后置处理器

SmartInstantiationAwareBeanPostProcessor

智能实例化已知的 Bean 后置处理器。

public interface SmartInstantiationAwareBeanPostProcessor extends InstantiationAwareBeanPostProcessor {
	@Nullable
	default Class<?> predictBeanType(Class<?> beanClass, String beanName) throws BeansException {
		return null;
	}
	@Nullable
	default Constructor<?>[] determineCandidateConstructors(Class<?> beanClass, String beanName)
			throws BeansException {

		return null;
	}
	default Object getEarlyBeanReference(Object bean, String beanName) throws BeansException {
		return bean;
	}
}
MergedBeanDefinitionPostProcessor

合并 Bean 定义对象的后置处理器。

public interface MergedBeanDefinitionPostProcessor extends BeanPostProcessor {
	void postProcessMergedBeanDefinition(RootBeanDefinition beanDefinition, Class<?> beanType, String beanName);
	default void resetBeanDefinition(String beanName) {}
}
SmartInitializingSingleton

智能初始化单例，只有一个方法，在所有的非懒加载单实例bean都成功创建并且放到Spring IOC容器之后进行执行的。

public interface SmartInitializingSingleton {
	void afterSingletonsInstantiated();
}
8个后置处理器的方法
1. InstantiationAwareBeanPostProcessor 的 postProcessBeforeInstantiation( )

时机：在 Bean 对象实例化之前。

作用：获取已知的 Bean 的实例对象，若成功获取，则这个返回值可以作为当前该生成的目标对象，即当前 Bean 实例，后续只需调用 5.和8. postProcessAfterInitialization 方法；否则按照正常的调用流程走。

2. SmartInstantiationAwareBeanPostProcessor 的 determineCandidateConstructors( )

时机：在 Bean 对象实例化之前。

作用：Spring 提供的 Bean 候选构造器接口，返回 Bean 候选构造器列表，允许开发者自定义决定使用哪个候选构造器。

实现类：AutowiredAnnotationBeanPostProcess。

实现：返回 Bean 候选的构造器接口，返回构造器中标注了@Autowired 注解的的方法，查找最适合创建 Bean 的构造器。

3. MergedBeanDefinitionPostProcessor 的 postProcessMergedBeanDefinition( )

时机：在 Bean 对象实例化之后。

作用：查找并缓存 Bean 注入信息，此时仍未完成注入。

4. SmartInstantiationAwareBeanPostProcessor 的 getEarlyBeanReference( )

时机：在 Bean 对象实例化之后。

作用：获得提前暴露的 Bean 引用，解决属性的循环依赖问题，只有单例对象才会调用此方法。

5. InstantiationAwareBeanPostProcessor 的 postProcessAfterInstantiation( )

时机：在 Bean 对象实例化之后。

作用：判断是否给 Bean 对象注入属性，如果该方法返回 false，会忽略属性值的设置；如果返回true，会按照正常流程设置属性值。

6. InstantiationAwareBeanPostProcessor 的 postProcessProperties( )

作用：Bean 属性赋值。

实现类：

AutowiredAnnotationBeanPostProcessor ：对 @Autowire @Value 标注的字段和方法注入值。

CommonAnnotationBeanPostProcessor ： 对 @Resource 标注的字段和方法注入值。

7. BeanPostProcessor 的 postProcessBeforeInitialization( )

时机：在 Bean 对象初始化之前。

实现类：

ApplicationContextAwareProcessor ：

如果 Bean 实现了EnvironmentAware、EmbeddedValueResolverAware、ResourceLoaderAware、ApplicationEventPublisherAware、MessageSourceAware、ApplicationContextAware 接口时，会调用其 postProcessBeforeInitialization( ) 方法将 Aware 前缀对应的对象注入到 Bean 对象中。

CommonAnnotationBeanPostProcessor ：调用其 postProcessBeforeInitialization( ) 方法，调用 Bean 中标注 @PostConstruct 注解的方法。

8. BeanPostProcessor 的 postProcessAfterInitialization( )

时机：在 Bean 对象初始化之后。

作用：判断该 Bean 对象是否需要 AOP 代理增强，如果需要的话，则会在该步骤返回一个代理对象。

9.SmartInitializingSingleton 的 afterSingletonsInstantiated( )

该方法会在所有的非懒加载单实例 Bean 都成功创建并且放到 Spring IOC 容器之后，依次遍历所有的 Bean，如果当前这个 Bean 是SmartInitializingSingleton 的子类，那么就强转成 SmartInitializingSingleton 类，然后调用 afterSingletonsInstantiated( ) 方法。

这个扩展点可以让我们自定义注解，完成我们想要实现的逻辑。
