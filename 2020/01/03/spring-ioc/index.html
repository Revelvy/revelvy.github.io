引言

Inversion of Control，控制反转，是一种设计思想。

⭐ 理解
控制反转：将 JavaBean 对象实例化的过程交给IOC容器（工厂）去负责，即将 JavaBean 对象的创建交给第三方。
反转：程序自身不创建对象，变为被动接收对象。
控制反转的实现方法：依赖注入，即为 JavaBean 对象的属性赋值，将 JavaBean 对象的属性注入到 IOC 容器，由 Spring 通过 XML 或注解的方式来创建、管理、装配对象。

⭐ 作用

  1. **解耦**：提高业务代码的聚合度，降低模块间的耦合度，即高内聚低耦合。
  2. **实例化 JavaBean 对象**。

核心
    工厂设计模式 （简单工厂 工厂方法 抽象工厂）
    XML解析 （Dom4j）
    反射技术 （实例化对象 反射获取方法 反射获取属性 反射获取构造器 反射调用方法）
    策略模式 （加载资源）
    单例（IOC创建实例化对象对象）

applicationContext.xml

  • xmlns ：xml 命名空间（namespace）
  • xmlns:xsi ：遵守规范（schema instance）
  • xsi:schemaLocation：本文档 xml 遵守的规范 官方指定
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
        https://www.springframework.org/schema/beans/spring-beans.xsd">
    <!--
        将对象放入 Spring IoC 容器中，进行实例化
            类型 对象名 = new 类型();
        id：等价于对象名 bean标签的唯一标识，对应的 JavaBean 对象名首字母小写
        class：等价于对象对应的全限定类型 JavaBean 对象的类路径 （包名 + 类名）
        name：别名 bean/对象的 id 别名，可以用别名获取对象，可多个
    -->
    <bean id="mysqlImpl" class="com.xxxx.dao.UserDaoMysqlImpl"></bean>
    <bean id="userService" class="com.xxxx.service.UserService" name="u2,u22">
        <!-- ref：引用Spring容器中已创建好的对象 -->
        <property name="userDao" ref="mysqlImpl" />
    </bean>
    <!-- 别名 bean/对象的id别名，可以用别名获取对象 -->
    <alias name="user" alias="user01" />
    <!-- 导入合并 -->
    <import resources="spring-dao.xml" />
    <import resources="spring-service.xml" />
</beans>

Test.java

public class Test {
    @test
    public static void main(String[] args) {
        // Spring上下文对象 / 配置文件 （单个配置文件 相对路径 快捷键CPX）
        ApplicationContext applicationContext = new ClassPathXmlApplicationContext("applicationContext.xml");
        // 从Spring的IoC容器中，通过id获取已经实例化好的对象
        UserService userService = (UserService) applicationContext.getBean("userService");
        // 调用对象的方法
        userService.test();
    }
}

手动注入

set方法（推荐）

/* set方法注入
    1. 成员属性字段提供set方法
    2. 配置文件中通过property属性指定成员属性字段
 */
public class UserService {
    // JavaBean B对象
    private UserDao userDao;
    public void setUserDao(UserDao userDao) {
        this.userDao = userDao;
    }
    // 基本数据类型
    private Integer port;
    public void setPort(Integer port) {
        this.port = port;
    }
    // 对象
    private String host;
    public void setHost(String host) {
        this.host = host;
    }
    // List集合
    private List<String> list;
    public void setList(List<String> list) {
        this.list = list;
    }
    // Set集合
    private Set<String> sets;
    public void setSets(Set<String> sets) {
        this.sets = sets;
    }
    // Map对象
    private Map<String, Object> map;
    public void setMap(Map<String, Object> map) {
        this.map = map;
    }
    // properties属性对象
    private Properties properties;
    public void setProperties(Properties properties) {
        this.properties = properties;
    }
    public void test() {
        System.out.println("UserService Test...");
        this.userDao.test();
        System.out.println(this.port);
        System.out.println(this.host);
        this.list.forEach((v -> System.out.println(v)));
        this.sets.forEach((v -> System.out.println(v)));
        this.map.forEach((k, v) -> System.out.println(k + "=" + v));
        this.properties.forEach((k, v) -> System.out.println(k + "=" + v));
    }
}
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
        https://www.springframework.org/schema/beans/spring-beans.xsd">
    <!--
        set方法注入：通过property属性指定成员属性字段，将JavaBean B对象注入到JavaBean A对象里
            name：JavaBean A 中对象的属性字段名
            ref：JavaBean B 对象的id
            value：具体的值（基本数据类型 对象 集合）
    -->
    <bean id="userService" class="com.xxxx.service.UserService">
        <!-- JavaBean对象（主要） -->
        <property name="userDao" ref="userDao"/>
        <!-- 基本数据类型 -->
        <property name="port" value="8080"/>
        <!-- 对象：String、Map、Properties -->
        <property name="host" value="127.0.0.1"/>
        <property name="map">
            <map>
                <entry>
                    <key><value>周杰伦</value></key>
                    <value>晴天</value>
                </entry>
                <entry>
                    <key><value>林俊杰</value></key>
                    <value>江南</value>
                </entry>
            </map>
        </property>
        <property name="properties">
            <props>
                <prop key="bj">北京</prop>
                <prop key="sh">上海</prop>
            </props>
        </property>
        <!-- 集合：List、Set -->
        <property name="list">
            <list>
                <value>上海</value>
                <value>北京</value>
            </list>
        </property>
        <property name="sets">
            <set>
                <value>上海SH</value>
                <value>北京BJ</value>
            </set>
        </property>
    </bean>
    <bean id="userDao" class="com.xxxx.dao.UserDao" />
</beans>
xmlns:p="http://www.springframework.org/schema/p"
    <!--
        p名称空间
             简化setter方法属性注入。
    -->
    <bean id="userService" class="com.xxxx.service.UserService"
          p:userDao-ref="userDao"
          p:port="8080"
          p:host="127.0.0.1"
    >

People.java

/**
 * 自动装配Bean对象（反射）
 * 1. 注解默认通过属性字段名称查找对应的bean对象（属性字段名称与bean标签的id属性值一致）
 * 2. 若属性字段名称不一样，则会通过类型（Class）类型
 * 3. 属性字段可提供set方法 也可不提供
 * 4. 注解可以声明在属性字段上 或 set方法级别
 * 5. 可设置注解的name属性，name属性值要与bean标签的id属性值一致（如果设置了name属性，则会按照name属性查询bean对象）
 * 6. 当注入接口时，若接口只有一个实现类，则不用指定需被实例化的bean对象；若接口有多个实现类，则需使用name属性指定
 */

/**
	衍生注解
		1.组件注解
			将类注册到Spring容器中代为管理
			约定
                Dao层		 @Repository 等价于 <bean id="people" class="com.xxxx.pojo.People" />
                Service层	  @Service
                Controller层	  @Controller
                任意类			@Component
		2.模式注解
			@Scope("singleton")  单例模式 等价于 <bean scope="singleton" id="people" />
			@Scope("prototype")  原型模式
*/
@Component
public class People {
    // 自动装配Bean对象（可不用写set方法 默认先使用byName找id，找不到就使用byType找类型）
    @Resource(name = "cat")
    private Cat cat;
    // @value 等价于 
    // <bean id="cat" class="com.xxxx.pojo.Cat">
    //      <property name="uname" value="名字" />
    // </bean>
    @Value("名字")
    public String uname;
    public void test() {
        cat.shout();
    }
}

作用域

单例作用域（默认）

Spring IOC 容器启动时，会将所有在单例作用域中的bean对象实例化放到单例缓存池中。

<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:context="http://www.springframework.org/schema/context"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
       https://www.springframework.org/schema/beans/spring-beans.xsd
       http://www.springframework.org/schema/context
       http://www.springframework.org/schema/context/spring-context.xsd">
        <!--
			singleton单例作用域
             lazy-init属性（懒加载）
                 如果设置为true，表示懒加载，容器在启动时，不会实例化bean对象，在程序调用时才会实例化
                 如果设置false（默认），表示不懒加载，容器启动则实例化bean对象
             lazy-init属性为什么要设置为false？
                  1. 可以提前发现潜在的配置文件
                  2. Bean对象在启动时就会设置在单例缓存池中，使用时不需要再去实例化bean对象，提高程序运行效率
             什么对象适合作为单例对象？（什么对象适合交给IOC容器实例化？）
                  无状态的对象（不存在改变当前对象状态的成员变量）如：controller层、service层、dao层
        -->
        <!-- 懒加载 -->
        <bean id="roleService" class="com.xxxx.service.RoleService" lazy-init="true"></bean>
        <!-- 默认false，不懒加载 -->
        <bean id="roleService" class="com.xxxx.service.RoleService" lazy-init="false"></bean>
</beans>

原型作用域

scope=”prototype”，Spring IOC 容器启动时，不会将bean对象实例化放到单例缓存池中，每次实例化对象都会创建一个新的实例。

<!-- propotype 原型作用域 -->
<bean id="roleService" class="com.xxxx.service.RoleService" scope="prototype"></bean>

Web应用中的作用域

1. request作用域
    表示每个请求需要容器创建一个全新Bean。比如提交表单的数据必须是对每次请求新建一个Bean
    来保持这些表单数据，请求结束释放这些数据。
2. session作用域
    表示每个会话需要容器创建一个全新Bean。比如对于每个用户一般会有一个会话，该用户的用户
    信息需要存储到会话中，此时可以将该Bean作用域配置为session级别。
3. globalSession作用域
    类似于session作用域，其用于portlet(Portlet是基于Java的Web组件，由Portlet容器管理，并由容器处理请求，生产动态内容)环境的web应用。如果在非portlet环境将视为session作用域。
    配置方式和基本的作用域相同，只是必须要有web环境支持，并配置相应的容器监听器或拦截器从而
    能应用这些作用域，目前先熟悉概念，后续集成web时讲解具体使用，大家只需要知道有这些作用域就
    可以了。

生命周期

  1. 定义
  • 在 xxxConfig 配置文件中定义bean（Java方式自动装配）。
  • 在 Applicaiton.xml 配置文件中通过bean标签定义对应bean对象。
  1. 初始化：IOC容器启动时，自动实例化Bean对象
  • 在配置文档中通过指定 init-method 属性来完成。
  • 实现 org.springframework.beans.factory.InitializingBean 接口。
  1. 使用
  • 使用Bean工厂对象：BeanFactory
  • 使用应用上下文对象： ApplicationContext
  1. 销毁
    步骤一：通过 AbstractApplicationContext 对象，调用其close方法实现bean的销毁过程。
    步骤二：在配置文件中指定对应销毁的方法 destroy-method。
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:context="http://www.springframework.org/schema/context"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
       https://www.springframework.org/schema/beans/spring-beans.xsd
       http://www.springframework.org/schema/context
       http://www.springframework.org/schema/context/spring-context.xsd">
        <!-- 初始化：在配置文档中通过指定 init-method 属性来完成-->
        <bean id="roleService" class="com.xxxx.service.RoleService" init-method="init"></bean>
        <!-- 初始化：实现 org.springframework.beans.factory.InitializingBean 接口 -->
        <bean id="roleService" class="com.xxxx.service.RoleService" ></bean>
        <!-- 销毁 -->
        <bean id="roleService" class="com.xxxx.service.RoleService" destroy-method="destroy" ></bean>
</beans>
