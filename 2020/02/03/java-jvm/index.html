Java JVM

Java JVM
Java

文章字数:   1.5k

引言

JVM 调优目标：方法区和堆。

架构

  • 线程私有：程序计数器、栈、本地⽅法栈。
  • 线程共享：⽅法区、堆、直接内存 (不在运⾏时数据区内)

类加载器

📌 ​作用：加载.class文件。

🌠 种类：

  1. 引导 类加载器 BootClassLoader ：JVM自带，负责装载核心类库。
  2. 扩展 类加载器 ExtClassLoader ：装载 jre/lib/ext 目录下的jar包。
  3. 应用 类加载器 AppClassLoader。
过程：
	1. 类加载：将.class文件的字节码加载到内存，并为之创建一个Class对象
	2. 类链接：将类的二进制代码合并到JRE
		验证
		准备：为 static变量 分配内存空间并设置初始值  m = 0
		解析：<clinit>(){ m = 100 }
	3. 类初始化
		时机：
			类的主动引用：
				虚拟机启动时，先初始化main方法所在的类
				new一个类的对象
				调用类的static成员（除了final）和static方法
				使用java.lang.reflect包的方法对类进行反射调用
				初始化类时，其父类没有被初始化，则优先初始化父类
			类的被动引用（不会发生类初始化）：
				通过数组定义类引用
				引用常量（已经在常量池中了）
				子类引用父类的静态常量
机制（双亲委派）
	1.从最内层JVM自带类加载器开始加载，不加载外层恶意同名类
	2.严格通过包区分访问域，外层恶意类通过内置代码也无法获取权限访问内层类
	  · 存取控制器：控制核心API对操作系统的存取权限
	  · 安全管理器：核心API和操作系统之间的接口，实现权限控制（优先级高）
	  · 安全软件包：java.security下的类和扩展包下的类
	  	  · 安全提供者
	  	  · 消息摘要
	  	  · 数字签名 keytools
	  	  · 加密
	  	  · 鉴别

Car car1 = new Car();
Car car2 = new Car();
car1.hashCode == car2.hashCode  // false
Class<? extends Car> clazz = car1.getClass()
ClassLoader classLoader = clazz.getClassLoader();
classLoader 或 ClassLoader.getSystemParent()  //AppClassLoader
classLoader.getParent()  //ExtClassLoader
classLoader.getParent().getParent()  //null 不存在或调用不到（C++写的 native）
System.getProperty("java.class.path")  //系统类加载器可以加载的路径

双亲委派

📌 作用：安全，防止恶意代码干涉正常代码。

过程：
    1.类加载器收到类加载的请求
    2.类加载器将请求向上委托给父类加载器去完成，一直向上委托，直到引导类加载器
    3.引导类加载器检查是否能加载当前类，能加载就结束，否则抛异常 ClassNotFound 通知自加载器进行加载

沙箱安全

📌 作用：将代码加入保护域，限制其运行环境，控制代码能进行哪些操作（限定在 JVM 特定的运行范围内，限制对本地系统资源的访问）。

组件：
	字节码检验器
	类加载器：
		防止恶意代码干涉正常代码【双亲委派机制】
		守护被信任的类库边界
		将代码归入保护域，限制其运行环境，控制代码能进行哪些操作【沙箱安全机制】

运行时数据区

方法区（特殊的堆）

Method Area 
概念： 被所有线程共享
存放： static变量 final变量 class变量 常量池

堆（存放对象的地址）

Heap
概念：一个 JVM 只有一个堆内存，可自由调节大小
存放：类、方法、常量、变量、所有new出来的引用类型的真实对象（被所有线程共享）
包含：
	新生区：类诞生、成长、死亡的地方
	养老区
	元空间（永久区）：
		常驻内存，存放 JDK 自身携带的Class对象，不存在垃圾回收，关闭 JVM 会释放此区域内存
		逻辑上存在，物理上不存在
long max = Runtime.getRuntime().maxMemory();  // JVM试图使用的最大内存
long total = Runtime.getRuntime().totalMemory();  // JVM总内存
max/(double)1024/1024 + "MB"
// OOM 堆内存溢出 报错解决方案

-Xms1024m -Xmx1024m -XX:+HeapDumpOnOutPutMemoryError  // OOM Dump

GC

分类：轻GC、重GC
作用 在：新生区、养老区
GC 算法：引用计数器、复制、标记清除、标记清除压缩

1. 引用计数器

2. 复制

优点：不产生内存碎片
缺点：需要额外内存空间（To必须为空），特别是极端情况：对象100%存活
场景：用于新生区（此时对象存活度低）

3. 标记清除

优点：不需要额外内存空间
缺点：两次扫描，浪费时间，会产生内存碎片

4. 标记清除压缩

优点：不需要额外内存空间，不产生内存碎片
缺点：三次扫描，浪费时间

内存效率：复制 > 标记清除 > 标记压缩
内存整齐：复制 = 标记压缩 > 标记清除
内存利用：标记压缩 = 标记清除 > 复制

GC分代收集算法：没有最好，只有最合适的算法
	新生区：用复制
	养老区：用标记清除 + 标记清除压缩

栈（存放对象的引用）

Stack
概念：栈内存，主管程序的运行、生命周期、线程同步，一旦线程结束，栈内存就释放栈Over，栈不存在垃圾回收
存放：八大基本数据类型、对象引用、对象方法

本地方法栈

Native Method Stack  登记本地方法（native method），在执行引擎执行时加载本地库（Native Libraries）
作用：调用底层C语言的库，进入本地方法栈，调用本地方法接口

程序计数器

Program Counter Register
概念：每个线程都有程序计数器（私有），就是指针，指向方法区中的方法字节码

本地方法接口

Java Native Interface（JNI）
概念：Java扩展，融合不同语言
